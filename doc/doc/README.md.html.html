<div class="pilwrap" id="generate-code">
  <h2>
    <a href="#generate-code" name="generate-code" class="pilcrow"></a>
Generate code
  </h2>
</div>
<p>If you happen to use Nx plugins, you can leverage code generators that might come with it.</p>
<p>Run <code>nx list</code> to get a list of available plugins and whether they have generators. Then run <code>nx list &lt;plugin-name&gt;</code> to see what generators are available.</p>
<p>Learn more about <a href="https://nx.dev/plugin-features/use-code-generators">Nx generators on the docs</a>.</p>
<div class="pilwrap" id="running-tasks">
  <h2>
    <a href="#running-tasks" name="running-tasks" class="pilcrow"></a>
Running tasks
  </h2>
</div>
<p>To execute tasks with Nx use the following syntax:</p>
<pre><code>nx &lt;target&gt; &lt;project&gt; &lt;...options&gt;
</code></pre>
<p>You can also run multiple targets:</p>
<pre><code>nx run-many -t &lt;target1&gt; &lt;target2&gt;
</code></pre>
<p>..or add <code>-p</code> to filter specific projects</p>
<pre><code>nx run-many -t &lt;target1&gt; &lt;target2&gt; -p &lt;proj1&gt; &lt;proj2&gt;
</code></pre>
<p>Targets can be defined in the <code>package.json</code> or <code>projects.json</code>. Learn more <a href="https://nx.dev/core-features/run-tasks">in the docs</a>.</p>
